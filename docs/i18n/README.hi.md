<h1>DsCode</h1>

[![][github-license-shield]][github-license-link]

**आपके टर्मिनल में AI कोडिंग असिस्टेंट।**

<br/>
</div>

**DsCode** एक टर्मिनल-आधारित AI कोडिंग असिस्टेंट है। आप एक AI मॉडल (जैसे DeepSeek V4) से बात करते हैं और वह आपके प्रोजेक्ट में कोड का विश्लेषण, सुझाव, समीक्षा और लेखन करता है। यह Windows, Linux और macOS पर काम करता है।

DsCode [DeepCode (lessweb/deepcode-cli)](https://github.com/lessweb/deepcode-cli) से लिया गया है और इसका अपना विकास है, जिसे [André Campos](https://github.com/andrelncampos) द्वारा बनाए रखा गया है।

---

## DsCode किसके लिए है

DsCode इनके लिए उपयोगी है:

- **डेवलपर्स** जो रोज़मर्रा के कार्यों में AI सहायता चाहते हैं।
- **टेक लीड्स** जिन्हें कोडबेस को जल्दी से समीक्षा या समझना है।
- **जो लोग पहले से AI से कोड कर रहे हैं** और एक तेज़, टर्मिनल-एकीकृत वर्कफ़्लो चाहते हैं।
- **टीमें जो मानकीकरण करना चाहती हैं** prompts, skills और agents को एकरूपता बनाए रखने के लिए।
- **DeepSeek V4 उपयोगकर्ता** जो thinking mode, reasoning effort और KV Cache का लाभ उठाना चाहते हैं।

| कार्य | DsCode कैसे मदद करता है |
|---|---|
| **कोडबेस का विश्लेषण** | पूछें "इस प्रोजेक्ट की आर्किटेक्चर समझाएं" और AI फ़ाइलें पढ़कर उत्तर देता है। |

| **सुविधाएं लागू करना** | बताएं कि आपको क्या चाहिए और AI फ़ाइलें जनरेट या संपादित करता है। |
| **रीफैक्टर** | पूछें "इस फ़ंक्शन को बिना व्यवहार बदले सरल बनाएं"। |
| **बग की जांच** | स्टैक ट्रेस पेस्ट करें और कारण खोजने में मदद मांगें। |
| **Skills बनाना या उपयोग करना** | Skills वे गाइड हैं जो AI को एक विशिष्ट तरीके से काम करना सिखाती हैं। |
| **Git के साथ काम** | AI ब्रांच, कमिट मैसेज सुझाता है और वर्ज़न किए गए बदलाव करता है। |
| **रीज़निंग कॉन्फ़िगर करना** | कठिन कार्यों के लिए *thinking mode* सक्षम करें — AI जवाब देने से पहले "सोचता" है। |
| **बाहरी टूल्स को एकीकृत करना** | MCP के साथ, डेटाबेस, ब्राउज़र, API और अन्य टूल्स कनेक्ट करें। |

| ऑपरेटिंग सिस्टम | डाउनलोड फ़ाइल |

| macOS (Intel x64) | `dscode-macos-x64.tar.gz` |
| macOS (Apple Silicon / ARM64) | `dscode-macos-arm64.tar.gz` |

प्रत्येक release में डाउनलोड अखंडता सत्यापित करने के लिए एक `checksums.txt` फ़ाइल शामिल है।

## ऑपरेटिंग सिस्टम के अनुसार इंस्टॉलेशन

### अनुशंसित इंस्टॉलेशन (कोई भी सिस्टम)

सबसे सरल तरीका npm के माध्यम से है:

```bash
npm install -g @andrelncampos/dscode
```

फिर किसी भी प्रोजेक्ट फ़ोल्डर में `dscode` चलाएं। यदि आपने Node.js इंस्टॉल नहीं किया है, तो नीचे पूर्वापेक्षाएं देखें।

**एकमात्र पूर्वापेक्षा**: [Node.js](https://nodejs.org) संस्करण **22** या उच्चतर।

अपना संस्करण जांचें:

```bash
node --version
```

आउटपुट `v22.x.x` या `v24.x.x` होना चाहिए। पुराने संस्करण (18, 20) समर्थित नहीं हैं।

### Windows

#### विकल्प 1: npm (अनुशंसित)

1. [Node.js 22+](https://nodejs.org) इंस्टॉल करें।
2. **PowerShell** (या Git Bash, Terminal, CMD) खोलें।
3. ग्लोबली इंस्टॉल करें:

   ```powershell
   npm install -g @andrelncampos/dscode
   ```

4. सत्यापित करें:

   ```powershell
   dscode --version
   ```

   संस्करण संख्या दिखनी चाहिए (जैसे: `1.0.1`)।

5. किसी भी प्रोजेक्ट में चलाएं:

   ```powershell
   cd C:\my-project
   dscode
   ```

#### विकल्प 2: Release बाइनरी

जब release उपलब्ध हों:

1. Releases पेज से `dscode-windows-x64.zip` डाउनलोड करें।
2. अपनी पसंद के फ़ोल्डर में निकालें (जैसे: `C:\dscode`)।
3. फ़ोल्डर को सिस्टम PATH में जोड़ें।
4. टर्मिनल में `dscode.exe` चलाएं।

#### सामान्य Windows समस्याएं

- **`npm install -g` के बाद "कमांड नहीं मिला"**: npm का PATH कॉन्फ़िगर नहीं हो सकता। टर्मिनल बंद करें और फिर से खोलें, या जांचें कि `%APPDATA%\npm` PATH में है।
- **इंस्टॉलेशन पर अनुमति त्रुटि**: PowerShell को एडमिन के रूप में चलाएं या npm का prefix स्थानीय फ़ोल्डर में कॉन्फ़िगर करें।

### Linux

#### विकल्प 1: npm (अनुशंसित)

1. [Node.js 22+](https://nodejs.org) इंस्टॉल करें (`nvm` या अपने वितरण के पैकेज मैनेजर का उपयोग करें)।

   ```bash
   # nvm के साथ उदाहरण
   nvm install 22
   nvm use 22
   ```

2. ग्लोबली इंस्टॉल करें:

   ```bash
   npm install -g @andrelncampos/dscode
   ```

3. सत्यापित करें:

   ```bash
   dscode --version
   ```

4. चलाएं:

   ```bash
   cd /path/to/project
   dscode
   ```

#### विकल्प 2: Release बाइनरी

जब release उपलब्ध हों:

1. `dscode-linux-x64.tar.gz` डाउनलोड करें।
2. निकालें:

   ```bash
   tar -xzf dscode-linux-x64.tar.gz
   ```

3. निष्पादन योग्य बनाएं (यदि आवश्यक हो):

   ```bash
   chmod +x dscode
   ```

4. PATH में ले जाएं:

   ```bash
   sudo mv dscode /usr/local/bin/
   ```

#### सामान्य Linux समस्याएं

- **ग्लोबली इंस्टॉल करते समय अनुमति अस्वीकृत (EACCES)**: npm का prefix स्थानीय निर्देशिका में कॉन्फ़िगर करें या `sudo` का उपयोग करें।
- **शेल `dscode` को नहीं पहचानता**: जांचें कि `/usr/local/bin` PATH में है। टर्मिनल फिर से खोलें।

1. [Node.js 22+](https://nodejs.org) इंस्टॉल करें (आधिकारिक इंस्टॉलर, Homebrew या nvm का उपयोग करें)।

2. ग्लोबली इंस्टॉल करें:

   ```bash
   npm install -g @andrelncampos/dscode
   ```

3. सत्यापित करें:

   ```bash
   dscode --version
   ```

4. चलाएं:

   ```bash
   cd /path/to/project
   dscode
   ```

#### विकल्प 2: Release बाइनरी

जब release उपलब्ध हों, अपने Mac के लिए सही फ़ाइल डाउनलोड करें:

- **Intel chip वाला Mac**: `dscode-macos-x64.tar.gz`

निकालें:


#### Gatekeeper के बारे में

macOS इंटरनेट से डाउनलोड किए गए बाइनरी के निष्पादन को ब्लॉक कर सकता है। यदि ऐसा होता है, तो आपको सिस्टम प्रेफरेंसेज में **सुरक्षा और गोपनीयता** में मैन्युअल रूप से अधिकृत करना होगा। **Gatekeeper को स्थायी रूप से अक्षम न करें** — केवल DsCode को अधिकृत करें।

## स्रोत कोड से इंस्टॉलेशन

उन लोगों के लिए जो नवीनतम विकास संस्करण चाहते हैं या योगदान करना चाहते हैं:

```bash
# 1. रिपॉजिटरी क्लोन करें
git clone https://github.com/andrelncampos/dscode.git
cd dscode

# 2. निर्भरताएं इंस्टॉल करें
npm ci

# 3. बिल्ड करें (typecheck + lint + format + bundle)
npm run build

# 4. स्थानीय लिंक बनाएं (dscode को ग्लोबली उपलब्ध कराता है)
npm link

# 5. सत्यापित करें
dscode --version
```

अब `dscode` आपके टर्मिनल में ग्लोबल कमांड के रूप में उपलब्ध है।

## प्रारंभिक सेटअप

DsCode अपनी कॉन्फ़िगरेशन `~/.deepcode/settings.json` (आपकी होम डायरेक्टरी में) से पढ़ता है। आप स्थानीय सेटिंग्स के लिए किसी विशिष्ट प्रोजेक्ट के अंदर `.deepcode/settings.json` भी रख सकते हैं।

### अपनी पहली कॉन्फ़िगरेशन बनाना

`~/.deepcode/settings.json` बनाएं:

```json
{
  "env": {
    "MODEL": "deepseek-v4-pro",
    "BASE_URL": "https://api.deepseek.com",
    "API_KEY": "अपनी_की_यहां_डालें"
  },
  "thinkingEnabled": true,
  "reasoningEffort": "max"
}
```

### API की कहां से प्राप्त करें

| प्रदाता | की कहां से प्राप्त करें |
|---|---|
| **DeepSeek** | [platform.deepseek.com](https://platform.deepseek.com) → API Keys |
| **OpenAI** | [platform.openai.com](https://platform.openai.com) → API Keys |
| **Anthropic** | [console.anthropic.com](https://console.anthropic.com) → API Keys |

### एनवायरनमेंट वेरिएबल्स के साथ कॉन्फ़िगर करना

`settings.json` के विकल्प के रूप में, आप एनवायरनमेंट वेरिएबल्स का उपयोग कर सकते हैं। DsCode `DEEPCODE_` उपसर्ग वाले किसी भी वेरिएबल को पहचानता है:

```bash
# Linux / macOS
export DEEPCODE_MODEL="deepseek-v4-pro"
export DEEPCODE_API_KEY="अपनी_की_यहां_डालें"

# Windows PowerShell
$env:DEEPCODE_MODEL = "deepseek-v4-pro"
$env:DEEPCODE_API_KEY = "अपनी_की_यहां_डालें"
```

### उपलब्ध कॉन्फ़िगरेशन विकल्प

| फ़ील्ड | प्रकार | विवरण | डिफ़ॉल्ट |
|---|---|---|---|
| `env.MODEL` | string | उपयोग करने के लिए AI मॉडल | `deepseek-v4-pro` |
| `env.BASE_URL` | string | प्रदाता का API बेस URL | `https://api.deepseek.com` |
| `env.API_KEY` | string | प्रदाता की API की | *(आवश्यक)* |
| `thinkingEnabled` | boolean | रीज़निंग मोड सक्षम करता है (AI जवाब देने से पहले "सोचता" है) | DeepSeek के लिए `true` |
| `reasoningEffort` | string | रीज़निंग गहराई: `"high"` या `"max"` | V4 Pro के लिए `"max"` |
| `temperature` | number | प्रतिक्रिया रचनात्मकता (0 से 2) | *(प्रदाता डिफ़ॉल्ट)* |
| `maxTokens` | number | प्रति प्रतिक्रिया टोकन सीमा | 65536 (Pro) / 32768 (Flash) |
| `debugLogEnabled` | boolean | डीबग लॉग `~/.deepcode/logs/` में सहेजता है | `false` |
| `permissions` | object | सूक्ष्म अनुमति नियंत्रण (पढ़ना, लिखना, नेटवर्क, आदि) | *(सभी अनुमत)* |
| `mcpServers` | object | MCP सर्वर कॉन्फ़िगरेशन | *(कोई नहीं)* |
| `notify` | string | प्रत्येक कार्य पूरा होने के बाद निष्पादित स्क्रिप्ट | *(कोई नहीं)* |
| `webSearchTool` | string | कस्टम वेब सर्च स्क्रिप्ट | *(बिल्ट-इन उपयोग करता है)* |

⚠️ **सुरक्षा**: अपना `settings.json` कभी किसी के साथ साझा न करें। इसमें आपकी API की होती है। DsCode का `.gitignore` पहले से `*.log` और `settings.json` को बाहर रखता है।

## 5 मिनट में पहला उपयोग

### चरण 1: इंस्टॉल करें

```bash
npm install -g @andrelncampos/dscode
```

### चरण 2: अपनी की कॉन्फ़िगर करें

अपनी API की और पसंदीदा मॉडल के साथ `~/.deepcode/settings.json` बनाएं (ऊपर कॉन्फ़िगरेशन अनुभाग देखें)।

### चरण 3: एक प्रोजेक्ट फ़ोल्डर खोलें

```bash
cd /path/to/your/project
```

यह कोई भी प्रोजेक्ट हो सकता है: Git रिपो, व्यक्तिगत प्रोजेक्ट, यहां तक कि खाली फ़ोल्डर भी।

### चरण 4: DsCode शुरू करें

```bash
dscode
```

आपको टेक्स्ट इनपुट फ़ील्ड के साथ एक स्वागत स्क्रीन दिखेगी। असिस्टेंट तैयार है।

### चरण 5: कुछ सरल पूछें

प्रॉम्प्ट फ़ील्ड में टाइप करें:

```
इस प्रोजेक्ट की संरचना को 3 वाक्यों में समझाएं।
```

**Enter** दबाएं। AI प्रोजेक्ट फ़ाइलों का विश्लेषण करेगा और उत्तर देगा।

### चरण 6: एक उपयोगी विश्लेषण मांगें

```
कोडबेस का विश्लेषण करें और बिना कुछ बदले संभावित सुधार बताएं।
```

AI कोड की जांच करेगा और सुधार सुझाएगा। यदि आवश्यक हो तो पूर्ण आउटपुट देखने के लिए `Ctrl+O` का उपयोग करें।

### चरण 7: समीक्षा और कमिट

जब AI फ़ाइलों में बदलाव करे, तो कमिट करने से पहले **प्रत्येक diff की समीक्षा करें**। DsCode दिखाता है कि क्या बदला गया और आप तय करते हैं कि स्वीकार करना है या नहीं।

> 💡 **टिप**: बड़े कार्यों का अनुरोध करने से पहले कमिट करें (`git commit`)। यदि कुछ गलत होता है, तो आप `git reset --hard` से पूर्ववत कर सकते हैं।

## व्यावहारिक उपयोग उदाहरण

नीचे प्रत्येक उदाहरण कुछ ऐसा है जिसे आप DsCode प्रॉम्प्ट फ़ील्ड में टाइप कर सकते हैं।

| कार्य | क्या टाइप करें |
|---|---|
| **आर्किटेक्चर समझना** | "इस प्रोजेक्ट की आर्किटेक्चर समझाएं, मुख्य मॉड्यूल क्या हैं और वे कैसे संवाद करते हैं।" |
| **बग ढूंढना** | "src/ में संभावित बग का विश्लेषण करें। केवल बताएं, कुछ भी न बदलें।" |
| **सुधार सुझाना** | "src/ में कोड के लिए प्रदर्शन और पठनीयता सुधार सुझाएं।" |
| **सुविधा लागू करना** | "src/form.ts में साइनअप फॉर्म में ईमेल सत्यापन जोड़ें।" |
| **रीफैक्टर** | "src/utils.ts में processData() फ़ंक्शन को बिना व्यवहार बदले स्पष्ट बनाने के लिए रीफैक्टर करें।" |
| **Diff की समीक्षा** | "पिछले कमिट के बदलावों की समीक्षा करें और समस्याएं बताएं।" |
| **टेस्ट बनाना** | "src/validators.ts में validateUser() फ़ंक्शन के लिए यूनिट टेस्ट बनाएं।" |
| **Skill का उपयोग** | "इस कोड का ऑडिट करने के लिए सुरक्षा समीक्षा skill का उपयोग करें।" |
| **AGENTS.md शुरू करना** | `/init` टाइप करें एक फ़ाइल बनाने के लिए जिसमें AI प्रोजेक्ट में पालन करने के निर्देश हों। |

DsCode **संवादात्मक रूप से** काम करता है: आप जो चाहिए वह टाइप करते हैं, AI जवाब देता है और टूल्स (फ़ाइलें पढ़ना, कमांड चलाना, कोड संपादित करना) का उपयोग करता है। आप प्रत्येक क्रिया की पुष्टि या अस्वीकार कर सकते हैं।

| **Permissions** | AI क्या कर सकता है इसका नियंत्रण: फ़ाइलें पढ़ना, लिखना, नेटवर्क एक्सेस, कमांड चलाना। | यदि आप प्रत्येक क्रिया को निष्पादन से पहले समीक्षा करना चाहते हैं तो प्रतिबंधात्मक अनुमतियां कॉन्फ़िगर करें। |
| **Workspace** | रूट फ़ोल्डर जहां DsCode चल रहा है। AI केवल इस फ़ोल्डर की फ़ाइलें देखता है (जब तक आप बाहरी एक्सेस अधिकृत न करें)। | DsCode को उस प्रोजेक्ट के रूट में खोलें जिस पर आप काम करना चाहते हैं। |
| **कम्पैक्शन** | जब बातचीत बहुत लंबी हो जाती है, DsCode टोकन सीमा में फिट होने के लिए इतिहास का सारांश बनाता है। | स्वचालित। यदि आप चाहें तो `/new` से नया सेशन बाध्य कर सकते हैं। |

## DeepSeek के साथ उपयोग

DsCode DeepSeek V4 मॉडल के लिए अनुकूलित है।

### समर्थित मॉडल

| मॉडल | सर्वोत्तम उपयोग | गति | लागत |
|---|---|---|---|
| `deepseek-v4-pro` | जटिल कार्य, आर्किटेक्चर, डीबगिंग, गहन तर्क | सामान्य | अधिक |
| `deepseek-v4-flash` | सरल कार्य, रीफैक्टरिंग, त्वरित समीक्षा | तेज़ | कम |

### DeepSeek कॉन्फ़िगरेशन

```json
{
  "env": {
    "MODEL": "deepseek-v4-pro",
    "BASE_URL": "https://api.deepseek.com",
    "API_KEY": "अपनी_की_यहां_डालें"
  },
  "thinkingEnabled": true,
  "reasoningEffort": "max"
}
```

### Thinking mode

*Thinking mode* AI को जवाब देने से पहले तर्क करने की अनुमति देता है। तर्क टोकन दिखाई देते हैं (प्रदर्शन मोड पर निर्भर करता है) और आप देख सकते हैं कि AI निष्कर्ष पर कैसे पहुंचा।

- **कब उपयोग करें**: गहन विश्लेषण की आवश्यकता वाले कार्य (आर्किटेक्चर, जटिल डीबगिंग, डिज़ाइन निर्णय)।
- **कब अक्षम करें**: सरल, तेज़ कार्य (छोटी रीफैक्टरिंग, सरल प्रश्न)।
- **प्रदर्शन नियंत्रण**: `/raw` का उपयोग करके पूर्ण तर्क दृश्य, सारांश या छिपे हुए के बीच टॉगल करें।

### Reasoning effort

- **`"max"`**: सबसे गहन तर्क। जटिल कार्यों पर V4 Pro के लिए आदर्श। अधिक टोकन खर्च करता है।
- **`"high"`**: अच्छा संतुलन। अधिकांश दैनिक कार्यों के लिए पर्याप्त।

### KV Cache (स्वचालित बचत)

DeepSeek संदर्भ के दोहराए गए भागों को कैश (KV Cache) में संग्रहीत करता है और कैश किए गए टोकन के लिए **शुल्क नहीं लेता**। लाभ उठाने के लिए:

- बातचीत की शुरुआत स्थिर रखें (system prompt, प्रारंभिक निर्देश)।
- बिना आवश्यकता के सेशन पुनः आरंभ करने से बचें — बातचीत बनाए रखने से लागत कम होती है।
- DsCode कैशिंग स्वचालित रूप से प्रबंधित करता है; आपको कुछ नहीं करना है।

### लागत संबंधी सावधानियां

- V4 Pro प्रति प्रतिक्रिया अधिक टोकन का उपयोग करता है। वास्तव में आवश्यक कार्यों के लिए उपयोग करें।
- V4 Flash सस्ता और तेज़ है। समीक्षा, रीफैक्टरिंग और रोज़मर्रा के कार्यों के लिए उपयोग करें।
- [DeepSeek प्लेटफ़ॉर्म](https://platform.deepseek.com) पर अपने उपयोग की निगरानी करें।

### DeepSeek सर्वोत्तम अभ्यास

1. रणनीतिक कार्यों के लिए `deepseek-v4-pro` और दैनिक कार्य के लिए `deepseek-v4-flash` का उपयोग करें।
2. `thinkingEnabled: true` रखें — तर्क गुणवत्ता में महत्वपूर्ण सुधार करता है।
3. यदि उत्तर काट दिया गया है, तो "जारी रखें" टाइप करें — AI वहीं से आगे बढ़ता है जहां रुका था।
4. बहुत बड़े प्रॉम्प्ट से बचें। विशिष्ट रहें कि किन फ़ाइलों का विश्लेषण करना है।

## सुरक्षा सर्वोत्तम अभ्यास

| **`settings.json` कभी कमिट न करें** | इसमें आपकी API की है। प्रोजेक्ट का `.gitignore` पहले से इसे बाहर रखता है, लेकिन दोबारा जांचें। |
| **अनुमति देने से पहले कमांड की समीक्षा करें** | AI शेल कमांड सुझा सकता है। पुष्टि करने से पहले पढ़ें, खासकर यदि `rm`, `sudo` या नेटवर्क शामिल हो। |

| **स्वीकार करने से पहले diff पढ़ें** | DsCode प्रत्येक बदलाव दिखाता है। समीक्षा करें — AI गलतियां कर सकता है। |
| **प्रॉम्प्ट में संवेदनशील डेटा पेस्ट न करें** | पासवर्ड, टोकन या ग्राहक डेटा जैसी जानकारी लॉग या उत्तरों में दिख सकती है। |
| **मदद मांगने से पहले लॉग साफ़ करें** | `~/.deepcode/logs/` में लॉग में कोड स्निपेट हो सकते हैं। साझा करने से पहले गोपनीय जानकारी हटाएं। |
| **प्रयोगों के लिए अलग ब्रांच का उपयोग करें** | बड़े बदलावों का अनुरोध करने से पहले `git checkout -b ai-experiment` बनाएं। यदि कुछ गलत होता है, तो ब्रांच को छोड़ दें। |

## समस्या निवारण (Troubleshooting)

| समस्या | संभावित कारण | कैसे हल करें |
|---|---|---|
| **`dscode: command not found`** | ग्लोबल npm PATH में नहीं है | टर्मिनल फिर से खोलें। Windows पर, `%APPDATA%\npm` जांचें। Linux/macOS पर, `~/.npm-global/bin` जांचें। |
| **`Node.js version not supported`** | Node संस्करण 22 से नीचे | [Node.js 22+](https://nodejs.org) इंस्टॉल या अपग्रेड करें। |
| **`npm ci` विफल** | निर्भरताएं असंगत | `node_modules` और `package-lock.json` हटाएं, फिर `npm install` चलाएं। |
| **401 त्रुटि (Unauthorized)** | API की गुम या अमान्य | जांचें कि `~/.deepcode/settings.json` या एनवायरनमेंट वेरिएबल में `API_KEY` सही है। |
| **429 त्रुटि (Too Many Requests)** | प्रदाता की दर सीमा पार | कुछ सेकंड प्रतीक्षा करें और पुनः प्रयास करें। प्रदाता के प्लेटफ़ॉर्म पर अपना प्लान जांचें। |
| **उत्तर काट दिया गया** | टोकन सीमा तक पहुंच गया | `settings.json` में `maxTokens` बढ़ाएं या फिर से शुरू करने के लिए "जारी रखें" टाइप करें। |
| **टाइमआउट / अत्यधिक देरी** | प्रदाता सर्वर ओवरलोड या नेटवर्क समस्या | प्रतीक्षा करें। यदि जारी रहे, तो मॉडल बदलें: अस्थायी रूप से Pro के बजाय Flash का उपयोग करें। |
| **Windows अनुमति त्रुटि** | npm के पास लेखन अनुमति नहीं | PowerShell को एडमिन के रूप में चलाएं या npm का prefix कॉन्फ़िगर करें। |
| **Linux/macOS अनुमति त्रुटि (EACCES)** | ग्लोबल npm के पास अनुमति नहीं | npm का prefix स्थानीय निर्देशिका में कॉन्फ़िगर करें या `sudo npm install -g` का उपयोग करें। |
| **`npm run build` विफल** | Typecheck या lint त्रुटि | त्रुटि पहचानने के लिए अलग-अलग कमांड चलाएं: `npm run typecheck`, `npm run lint`, `npm run bundle`। |
| **लॉग दिखाई नहीं दे रहे** | `debugLogEnabled` `false` है (डिफ़ॉल्ट) | `settings.json` में `"debugLogEnabled": true` सक्षम करें। लॉग `~/.deepcode/logs/debug.log` पर दिखाई देंगे। |
| **मॉडल पहचाना नहीं गया** | गलत मॉडल नाम | सटीक नामों का उपयोग करें: `deepseek-v4-pro`, `deepseek-v4-flash`, या एक मान्य OpenAI-संगत मॉडल। |
| **टोकन खपत बहुत अधिक** | लंबा संदर्भ या बहुत व्यापक कार्य | सेशन रीसेट करने के लिए `/new` का उपयोग करें। फ़ाइलों और दायरे के बारे में विशिष्ट रहें। पूरे प्रोजेक्ट का विश्लेषण करने के लिए न कहें। |
| **बड़े रिपॉजिटरी में त्रुटि** | अनदेखी फ़ाइलें छोड़ी नहीं जा रहीं | DsCode `.gitignore` का सम्मान करता है। जांचें कि आपका `.gitignore` सही है। |

## सहायता कैसे प्राप्त करें

यदि आपको कोई समस्या आती है, तो [GitHub पर एक issue खोलें](https://github.com/andrelncampos/dscode/issues)।

रिपोर्ट करते समय, शामिल करें:

- **DsCode संस्करण**: `dscode --version`
- **ऑपरेटिंग सिस्टम**: Windows 11, Ubuntu 24.04, macOS 15 आदि
- **Node.js**: `node --version`
- **उपयोग किया गया मॉडल**: `deepseek-v4-pro`, `deepseek-v4-flash` आदि
- **निष्पादित कमांड** और पूरी त्रुटि
- **साफ़ किए गए लॉग**, यदि प्रासंगिक हों (की, टोकन और निजी डेटा हटाएं)

⚠️ **कभी न भेजें**:
- API की या टोकन
- आपके निजी प्रॉम्प्ट या गोपनीय प्रोजेक्ट डेटा
- पूर्ण `.env` या `settings.json` फ़ाइलें
- बिना समीक्षा के पूर्ण लॉग (उनमें कोड स्निपेट होते हैं)

सुरक्षा कमजोरियों के लिए, [SECURITY.md](../../SECURITY.md) में दिए गए निर्देशों का पालन करें। **सुरक्षा खामियों के लिए सार्वजनिक issue न खोलें।**

योगदान का स्वागत है! [CONTRIBUTING.md](../../CONTRIBUTING.md) में पूरी गाइड देखें।

त्वरित सारांश:

3. **सुरक्षा PR** या संवेदनशील क्षेत्रों में बदलाव की अधिक कठोर समीक्षा होती है।
4. योगदानकर्ता घोषित करते हैं कि उनके पास सबमिट किए गए कोड का योगदान करने का अधिकार है।
